Detailed Description

Handle of a PID component.

This structure holds all the parameters needed to implement a proportional-integral-derivative controller function. It also stores upper and lower limits used to saturate the integral term and the output values of the controller.

A pointer on a structure of this type is passed to each function of the PID Regulator component.

◆ hKdDivisor

uint16_t PID_Handle_t::hKdDivisor

\(K_{dd}\), divisor of the derivative gain gain

Used in conjuction with \(K_{dg}\), the derivative gain numerator to allow for obtaining fractional gain values.

If #FULL_MISRA_C_COMPLIANCY is not defined the divisor is implemented through algebrical right shifts to speed up the execution of the controller functions. Only in this case does this parameter specify the number of right shifts to be executed.

◆ wIntegralTerm

int32_t PID_Handle_t::wIntegralTerm

integral term of the PID Regulator

This value is updated on each call to the PI_Controller() or PID_Controller() functions. It contains the integral term before being divided by \(K_{id}\), the divisor of the integral gain:

\[ K_{ig}\times\sum_{k=0}^t{\epsilon_k} \]

This field is reset to 0 when the component is initialized.

See also
PID_HandleInit().
